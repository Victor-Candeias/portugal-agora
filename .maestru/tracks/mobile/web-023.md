# WEB-023: Serviços Públicos: PSP/GNR via Overpass (SQLite/WASM) + página perto de mim

Fonte: `.Docs/Justica.txt` — documento propõe três categorias de serviços públicos (Lojas do
Cidadão, Tribunais, Polícia PSP/GNR/Municipal/Marítima), com arquitetura comum: fontes oficiais
combinadas com OpenStreetMap, processadas em GitHub Actions para SQLite/GeoJSON estático,
publicado no GitHub Pages e consultado no browser via SQLite compilado para WebAssembly
(mesmo padrão já usado para Carris e Metro do Porto — WEB-010/011/016/022).

Este work-item implementa a **Fase 1** recomendada explicitamente no documento ("a mais
simples: Overpass API → normalização → GeoJSON/SQLite → mapa → pesquisa perto de mim"):
esquadras PSP, postos GNR, polícia municipal e polícia marítima, via Overpass
(`amenity=police`, nacional).

## Implementação

- `apps/web/scripts/build-public-services-db.mjs` — consulta a Overpass API
  (`overpass-api.de`) por `node`/`way`/`relation` com `amenity=police` em Portugal
  (`area["ISO3166-1"="PT"][admin_level=2]`), classifica cada local em
  `police_psp`/`police_gnr`/`police_municipal`/`police_maritime`/`police_other` a partir de
  `name`/`operator`/`brand`/`police` (regras exatas do documento) e gera
  `apps/web/public/data/public-services.sqlite` com o esquema `public_services` proposto
  (incluindo `confidence`: 70 com operador conhecido, 50 sem). Testado: 946 locais
  (261 PSP, 566 GNR, 22 Municipal, 24 Marítima, 73 outras) a partir de 1000 elementos OSM.
- `apps/web/package.json` — `build:db` passou a correr também este script (reutiliza o
  workflow diário existente em `.github/workflows/deploy.yml`, sem necessidade de novo cron).
- `apps/web/src/lib/publicServicesDb.ts` — carregamento do `.sqlite` via `sql.js`/WASM,
  mesmo padrão de `metroPortoDb.ts`, instância separada (domínio de dados independente).
- `apps/web/src/hooks/usePublicServices.ts` — `usePublicServices`, `usePublicServicesMeta`.
- `apps/web/src/pages/ServicosPublicos.tsx` — pesquisa por nome/localidade, filtro por
  categoria, geolocalização do utilizador ("perto de mim", mesmo padrão de
  `Hospitais.tsx`/WEB-006: `navigator.geolocation` com ordenação por distância e fallback
  manual), detalhes por local (morada, telefone, email, horário) e mapa opcional
  (reutiliza `SinglePointMap`, sempre visível junto ao item quando há morada/coordenadas —
  sem passo de "Detalhes" intermédio, tal como em `Hospitais.tsx`/`Turismo.tsx`). Registada
  em `main.tsx` (rota `/servicos-publicos`) e `Layout.tsx` (novo item de navegação
  "Serviços Públicos").

## Não implementado (fora deste WI)
- **Tribunais** (Fase 2 do documento) — requer scraping do Portal da Justiça /
  `justicanomapa.justica.gov.pt` (sem API pública conhecida); não implementado nesta fase.
- **Lojas do Cidadão** (Fase 3 do documento) — requer processamento de XLSX mensal do
  dados.gov.pt + enriquecimento gov.pt/OSM; não implementado nesta fase.
- Deduplicação com fontes oficiais/municipais do dados.gov.pt (o documento sugere combinar
  fontes; esta fase usa apenas OpenStreetMap, única fonte com cobertura nacional imediata).
- App mobile (Expo) — cobre apenas a app web; o mesmo padrão pode ser replicado depois.

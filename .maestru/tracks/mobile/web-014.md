# WEB-014: Carris: localização de veículos em tempo real

## Contexto

Fase 8 do `.Docs/CarrisDev.txt`: `GET /v2/vehicles`, atualização 5–10s (posição GPS, velocidade, direção, pattern, trip, matrícula, operador quando disponível).

**Validação:** já existe `useCarrisVehicles` em `useCarris.ts` a consumir `/v2/vehicles` diretamente do cliente (`refetchInterval: 30s`). **Mantém-se fetch direto** à API oficial — dados dinâmicos não vão para o SQLite. A melhoria é enriquecer a resposta em runtime com o `operator` (lookup local ao `.sqlite`, tabela `operators`/`routes` de WEB-016), sem pedido de rede extra.

## Objetivo

- Manter polling direto a `GET /v2/vehicles` (sem persistência).
- Após WEB-016, fazer join local (WASM) `line_id`/`route_id` → `operator` para mostrar o operador de cada veículo sem round-trip adicional.

## Depende de

WEB-010 (modelo de tipos comum). Relacionado com WEB-016. Bloqueia WEB-017.

# WEB-013: Carris: chegadas em tempo real com cache de curta duração

## Contexto

Fase 7 do `.Docs/CarrisDev.txt`: `GET /v2/arrivals/by_stop/{stopId}`, atualização a cada 15s.

**Validação:** já implementado no cliente em `useStopRealtime` (`useCarris.ts`, `refetchInterval: 30s`, `staleTime: 15s`) a chamar a API oficial diretamente. **Não faz sentido guardar isto no SQLite estático** — mantém-se fetch direto à API oficial, tal como hoje. Este WI passa a ser sobre uniformizar/reaproveitar o modelo de tipos comum (WEB-010) e, se aplicável, resolver `stopId`/`lineId` por lookup local ao `.sqlite` (WEB-011) antes de pedir os dados dinâmicos.

## Objetivo

- Manter fetch direto a `GET /v2/arrivals/by_stop/{stopId}` (React Query, cache curta ~15s em memória, nunca persistido).
- Usar o `.sqlite` local apenas para resolver metadados da paragem/linha (nome, coordenadas) antes/depois da chamada em tempo real.

## Depende de

WEB-010 (modelo de tipos comum). Bloqueia WEB-017.

# WEB-021: Turismo: enriquecimento on-demand via Wikidata (descrição + foto Wikimedia Commons)

Fonte: `.Docs/turismo.v2.txt`, que sugere Wikidata (descrições/metadados) e Wikimedia Commons
(fotografias) para enriquecer os pontos de interesse já integrados (SIGTUR/ICNF/UNESCO).

## Contexto

WEB-020 tinha descartado Wikidata/Wikimedia "por agora" por ser um problema N+1 (1 pedido extra
por cada um dos centenas de pontos já listados, sem cache de servidor — app 100% client-side).
Esta limitação só se aplica a enriquecimento em bloco de toda a listagem. Este WI implementa o
enriquecimento **on-demand**: 1 pedido (busca + entidade) só quando o utilizador expande
explicitamente um ponto individual ("Saber mais"), o que elimina o problema N+1.

## Implementação

- `packages/core/src/api/wikidata.ts` — `getWikidataEnrichment(name)`: usa
  `action=wbsearchentities` (Wikidata Query Service REST) para encontrar o QID pelo nome do
  ponto, depois `action=wbgetentities` para obter descrição (pt, fallback en), ligação para a
  Wikipédia (pt/en) e fotografia (propriedade `P18`, servida via
  `commons.wikimedia.org/wiki/Special:FilePath/<ficheiro>`).
- CORS: confirmado que a API MediaWiki do Wikidata devolve
  `Access-Control-Allow-Origin: *` quando se usa o parâmetro `origin=*` — não é necessário
  proxy/JSONP como no SIGTUR (WEB-018). As imagens via Special:FilePath funcionam diretamente
  como `<img src>`/`Image source`, sem necessitar de CORS.
- `apps/web/src/pages/Turismo.tsx` — extraído `TourismPointCard`, com botão "Saber mais" que
  ativa `useWikidataEnrichment` (React Query, `enabled` só quando expandido, `staleTime` 24h).
  Mostra descrição + fotografia + link para a Wikipédia.
- `apps/mobile/hooks/useApi.ts` — `useWikidataEnrichment`; `apps/mobile/app/(tabs)/turismo.tsx`
  — mesmo padrão, `TourismPointCard` com estado `showInfo` local por cartão.
- `packages/core/src/index.ts` — export do novo módulo `wikidata.ts`.

## Não implementado (fora deste WI)

- **DGPC** (património classificado): o site `patrimoniocultural.gov.pt` devolve HTTP 403 a
  pedidos automatizados (testado), sem API REST pública alternativa identificada — não
  integrado.
- **APIs municipais** (eventos/festas populares): cobertura muito heterogénea por município,
  sem uma fonte nacional única — não avaliado neste WI.

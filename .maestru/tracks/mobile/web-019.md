# WEB-019: Turismo: integrar ICNF (Áreas Protegidas, Rede Natura 2000, Percursos Pedestres)

## Implementação

Fonte real e funcional (confirmada via catálogo ArcGIS Online, item "Planos de Ordenamento
das Áreas Protegidas (POAP)", owner `admin.esri`): `https://sigservices.icnf.pt/server/rest/services/BDG`.
CORS aberto para qualquer origem (reflete o `Origin` do pedido) — não foi necessário JSONP/proxy
como no SIGTUR; a app web usa `fetch` direto tal como a mobile.

Layers integradas:
- `RNAP/FeatureServer/0` — Rede Nacional de Áreas Protegidas (Parques Nacionais/Naturais,
  Reservas Naturais, Paisagens Protegidas, Monumentos Naturais, Áreas Protegidas Privadas) →
  categoria `protected-areas`, subcategoria = campo `classifica`.
- `RN2000/FeatureServer/0` (ZPE) e `RN2000/FeatureServer/1` (SIC) → categoria `natura-2000`.
- `percursos_pedestres/FeatureServer/0` → categoria `trails`.

Como estas geometrias são polígonos/linhas (não pontos), reutiliza-se o modelo `TourismPoint`
já existente (SIGTUR) calculando o centro do bounding box da geometria como localização do
marcador (`packages/core/src/api/icnf.ts` → `boundingBoxCenter`).

Novo pacote `packages/core/src/api/tourism.ts` expõe `tourismClient`, que agrega
`sigturClient` + `icnfClient` num único cliente; `apps/web/src/pages/Turismo.tsx`,
`apps/mobile/hooks/useApi.ts` e `apps/mobile/app/(tabs)/turismo.tsx` foram atualizados para
usar `tourismClient` em vez de `sigturClient` diretamente, e os mapas `CATEGORY_LABEL` de
ambas as apps ganharam as novas categorias.

`Locais_ICNF` (delegações/postos ICNF) e "centros de interpretação" dedicados não foram
integrados: o catálogo ICNF não expõe uma layer específica de centros de interpretação — o
mais próximo (`Locais_ICNF`) contém delegações regionais e postos administrativos, sem
distinção clara de "centro de interpretação", pelo que foi excluído para evitar dados
enganosos.

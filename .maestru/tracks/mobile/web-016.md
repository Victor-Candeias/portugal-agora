# WEB-016: Carris: identificação de operadores via GTFS (agency.txt + routes.txt)

## Contexto

A API da Carris Metropolitana não tem endpoint dedicado a operadores. Estratégia do `.Docs/CarrisDev.txt`: usar `GET /v2/gtfs`, extrair `agency.txt` (id, nome, website, timezone) e `routes.txt` (relação rota→operador) para determinar o operador de cada linha/veículo (ex.: linha 4705 → Alsa Todi; linha 1710 → Rodoviária de Lisboa).

**Validação:** já existe uma UI parcial em WEB-007 ("combobox operador e carreira") — confirmar de onde vem hoje essa informação de operador no código atual antes de duplicar lógica; provavelmente precisa de ser substituída pela leitura real do GTFS em vez de heurística no frontend.

## Objetivo

- Job de sync diário do GTFS (agency.txt + routes.txt) integrado no mesmo script de build que gera o `.sqlite` (WEB-011) — tabela `operators` + coluna `operator_id` em `lines`/`routes`.
- Sem endpoint/servidor: o atributo `operator` fica disponível diretamente por query local ao `.sqlite` no cliente.

## Depende de

WEB-010, WEB-011 (mesmo pipeline de build). Relacionado com WEB-014, WEB-017.
